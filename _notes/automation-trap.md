The most expensive automation projects I've seen all started the same way: a messy manual process, automated as-is, because the mess was invisible while humans were quietly absorbing it. Remove the humans and the mess executes at machine speed — wrong data propagated instantly, edge cases mishandled identically every time, errors duplicated faster than anyone can catch them.

The rule: fix, then automate. Map the process, kill the workarounds, define the exceptions — and only then wire it up. Automation is an amplifier. Make sure what you're amplifying deserves it.
